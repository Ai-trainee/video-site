该错误通常与网络问题或远程服务器问题有关。以下是一些可能的解决方案：

  1. 网络问题: 确保网络连接稳定。
  2. 代理设置: 如果您在使用代理服务器，请检查代理设置。
  3. 重新认证: 运行git credential reject "https://github.com"，然后再次尝试git push。
  4. 增加Git缓冲区: 运行git config http.postBuffer 524288000。
  5. 使用SSH而非HTTPS: 更改远程URL为SSH格式。
  6. 服务器问题: 如果以上都不行，可能是GitHub服务器的问题。稍后再试或联系GitHub支持。

您可以先从上面的列表中尝试一两个解决方案，看是否能解决问题。

我的确是使用代理的

如果您正在使用代理，以下是一些可能的解决方案：

  1. 配置Git代理: 使用命令git config --global http.proxy [代理地址]和git config --global https.proxy [代理地址]来设置Git代理。
  2. 检查代理设置: 确保代理服务器能够访问GitHub，并且没有任何流量限制或阻断。
  3. 使用SOCKS代理: 如果您的代理是SOCKS代理，可以使用git config --global http.proxy 'socks5://[代理地址]'。
  4. 取消代理: 临时取消代理设置，然后尝试git push。
  5. 代理认证: 如果代理需要认证，确保您已经正确输入了凭据。

现在我要在本地新建一个文件夹，名叫video，怎么推送给仓库

  1. 创建文件夹: 在本地仓库中创建一个名为video的新文件夹。

    bash
    mkdir video

  2. 添加文件: 如果您想在video文件夹中添加一些文件，可以现在做。

  3. Git添加: 将新文件夹添加到Git跟踪。

    bash
    git add video/

  4. 提交更改: 提交您的更改。

    bash
    git commit -m "添加video文件夹"

  5. 推送到远程仓库: 将您的更改推送到远程仓库。

    bash
    git push origin main

出现这个问题通常是因为SSH密钥没有正确设置或没有添加到GitHub账户。以下是解决方案：

  1. 检查SSH密钥: 运行ssh-add -l检查是否有可用的SSH密钥。如果没有，您需要生成一个新的SSH密钥。

    bash
    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

  2. 添加SSH密钥到SSH-Agent:

    bash
    ssh-add ~/.ssh/id_rsa

  3. 添加SSH密钥到GitHub账户: 将SSH密钥（通常位于~/.ssh/id_rsa.pub）的内容复制到GitHub账户的SSH设置中。

  4. 测试连接:

    bash
    ssh -T git@github.com

  5. 推送代码:

    bash
    git push origin main

按照这些步骤操作，应该能解决问题。如果问题依然存在，您可能需要检查是否有多个SSH密钥或其他特殊配置导致的问题。
